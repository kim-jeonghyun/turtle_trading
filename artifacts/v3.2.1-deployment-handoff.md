## 2. 배포 실행

```bash
bash scripts/deploy-v3.2.1.sh
```

스크립트가 자동으로 다음을 수행합니다.

1. `data/positions/positions.json` 백업 생성
2. 기존 Docker 이미지에 `previous` 태그 부여
3. 신규 이미지 빌드 (`turtle-trading:v3.2.1`)
4. `signal_check` 참조 유무 최종 검증
5. 컨테이너 재시작 (`docker-compose up -d`)

세부 동작은 `scripts/deploy-v3.2.1.sh` 를 참조하세요.

## 5. 이상 발생 시 대응

### 롤백 기준

다음 중 하나라도 해당되면 즉시 롤백을 검토하세요.

- 반복적인 예외(Exception) 발생
- Telegram 알림 2회 연속 미수신
- `positions.json` 손상 또는 쓰기 실패
- fill 매칭 오판 (체결 가격/수량 불일치)

### 상세 롤백 절차

`.sisyphus/reviews/v3.2.1-rollback-procedure.md` 를 참조하세요.

### 빠른 롤백 명령 (긴급 시)

```bash
cd /Users/momo/dev/turtle_trading

# positions.json 사전 백업
BACKUP_TS=$(date +%Y%m%d_%H%M%S)
cp data/positions/positions.json \
   data/positions/backups/positions.json.pre-rollback-${BACKUP_TS}

# 컨테이너 중단
docker-compose down

# 이전 커밋(v3.2.0)으로 롤백 브랜치 생성
git checkout -b rollback/v3.2.1-to-v3.2.0 238760b

# 이전 버전 이미지 빌드 및 태깅
docker build -t turtle-trading:rollback-v3.2.0 .
docker tag turtle-trading:rollback-v3.2.0 turtle-trading:latest

# 컨테이너 재시작
docker-compose up -d
```
